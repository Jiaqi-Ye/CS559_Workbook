Here is a concise summary of the key takeaways for students, based on the workbook’s content:

The Big Idea: Manipulating the “Graph Paper” The most critical concept is that transformations (translation, rotation, scale) are best understood as moving the coordinate system itself—shifting, spinning, or stretching the “graph paper”—rather than moving individual points. This intuition is the foundation for understanding why the order of operations matters and how complex, articulated objects (like robots) are built by chaining these coordinate moves together.

Key Takeaways:

  • Coordinate Systems: Always think about transformations as modifying the “current” coordinate system. For example, translate moves the origin of the grid you are about to draw on.
  • Order Matters: Transformations are non-commutative. A rotate followed by a translate produces a different result than a translate followed by a rotate because each step alters the reference frame for the next.
  • Hierarchical Modeling: Complex objects are built using parent-child relationships. A “child” part (like a propeller) inherits the coordinate transformations of its “parent” (the quadcopter body), allowing them to move together naturally.
  • State Management: In Canvas, you must use save() and restore() to isolate transformations so that moving one object doesn’t accidentally move the next one you draw.
  • The Math: Under the hood, these 2D affine operations are represented as 3x3 matrices using homogeneous coordinates, which allows translation to be treated as a linear operation.
  • Retained Mode API: we saw SVG, a retained mode API. It represents the scene graph explicitly.
