Page 7: Mesh Exercise

In the world below (files 05-07-01.html and 05-07-01.js ) make (at least) two objects and place them into the world so they can be seen.

They can be anything. They must have at least 3 or more triangles, and you have to make each one from a BufferGeometry and put them into the 559 Framework. You can even use the objects that you’ll make for later parts of the assignments if they fulfill the requirements.

  1. One object must have face coloring where we can see triangles with different colors. With BufferGeometry you need to split the vertices. Each triangle should be one color and triangles with different colors should share an edge.
  2. One object must have vertex coloring where we can see the colors smoothly varying across a triangle boundary.
  3. One object must have vertex normals set so the lighting is smooth across a triangle boundary.

There are examples of face coloring, vertex coloring, and fake normals to get smooth lighting on previous pages. You can look at that code to understand how to make these different kinds of things. For this page, you cannot use texturing (that will come soon enough).

You may not use an object loader for this exercise: you must build the BufferGeometry object in JavaScript code. You need to actually have the coordinates in your file (yes, you need to figure out the vertex positions, and have the triangles oriented correctly). It is okay to use computeVertexNormals.

The other requirement is that it should be visibly obvious that you have met the requirements. The smooth normals might require rotating the object to see them.

This page is meant to be practice for the bigger exercises later.
