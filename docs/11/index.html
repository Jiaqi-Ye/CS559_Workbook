Page 11: Using Multiple Maps

Modern graphics use a variety of texture mapping tricks to create the appearances of objects. This means that for any point on an object, many different texture maps may be involved.

We can categorize these into two rough categories:

  • Maps that provide information about the specific point. This is the classic texture mapping application where each point on an object has a texture (uv) coordinate that specifies a location to lookup in a texture. Points could have different coordinates for different maps. Regular (color) texture maps and bump/normal maps are examples of these textures.
  • Maps that provide information about the scene. Environment maps and shadow maps are examples. For these, the texture coordinates are usually computed dynamically based on where the object is.

Advanced Material Maps

Controlling object appearance with texture maps allows each point on the object to have different properties. The idea is that there is a single “super-material” that supports many different appearances, and that these appearances are controlled per-point using parameters from the texture map. The alternative is to use different shading algorithms for each triangles, which is problematic because switching between materials is costly (it requires switching shader programs).

The standard maps used for appearance include:

  • Color (albedo) - this is the primary color the point appears. It is used for diffuse shading, and can be used for metalic highlights. This is what “normal texture maps” do.
  • Bump/Normal - this changes the direction of the normal directions to make the surface look less smooth.
  • Metalness - this controls the color of the specular highlights. Metallic objects reflect their primary color; plastic objects have white specular highlights.
  • Roughness - this controls how shart the specular highlights are, roughly what the specular exponents does in the Phong model. It also controls the amount of specular reflection.
  • Ambient Occlusion - this controls how much light gets to the point, simulating the effect of self shadowing.

The idea is that an object is one mesh that has UVs set to connect points on the object to appropriate places in the maps (plural) that switch between the varied properties.

The ambient occlusion map is typically computed by running an lighting calculation that computes detailed shadows. This precomputation is only for self-occlusions (or limited inter-object occlusions), but can add a lot of appearance detail.

You can see a demo with some complex surface appearances that use multiple maps (and a tool that lets you turn the maps on and off) at the AmbientCG Viewer. AmbiegCG is a website that has many texture examples, including surface properties.

Layers

An object might use multiple textures to “layer” appearances. For example, the main maps might provide a shiny metal or wood surface and a layer might add dirt or other imperfections.

Summary: Multi-Maps

The computation of the color for a single point can involve a lot of different maps. But they allow a consistent process to be applied to all points on an object which avoids switching costs.
