Page 10: Texturing Hacks

While the graphics pipeline is fast, it is limiting. The restriction that each triangle and fragment must be independent means that many effects that require interdependence of locations in the world (either multiple triangles, or multiple objects) cannot be supported. That means there is no direct way to create effects such as shadows and reflections.

What the rasterization pipeline does give us is a flexible set of abstractions from which we can build things:

  • We can perform computations on each vertex, provided it is independent from other vertices.
  • We can perform computations on each fragment, provided it is independent from other fragments.
  • These computations can access “common” data, such as the positions of a set of lights.
  • These computations can access textures.
  • We have the ability to read from and write to the target image (the frame buffer).
  • We can run the rasterization process multiple times to create a single image. This is often called multi-pass rendering, since we make “multiple passes” over the objects. With each pass, we draw the objects in a manner that comes together into the final result.

Over the years, graphics programmers have created a set of techniques that allow us to simulate various effects by putting these flexible abstractions together in different ways. They are “hacks” - in that they are clever, heuristic methods that fake the effects, rather than simulating them directly.

We’ve seen some of these already when we explored texture mapping. Reflection mapping (using an environment map) allowed us to simulate basic reflections. On the next pages, we will see some techniques that more directly create interdependent effects using similar tricks.
