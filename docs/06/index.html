Page 6: Fake Normals

On the previous pages, we discussed places where we choose the normals for triangles to be different from what the actual geometry of the triangle would suggest.

Here, we’ll show some applications of this. These will become even more important when we add lighting and reflection that depend on the normals.

Smooth-looking objects

Here is a scene with a bunch of cylinders. In the front row, the circular shape is approximated with six sides, but this is shown with both “real normals” (so you can see the faces) and “fake normals” (vertex normals computed so they match the real cylinder). The vertex normals are interpolated.

texturedemos/normals/n1-cylindernormal.html

If you don’t look at the ends of the cylinder (where the shape is obvious), even the six-sided one looks smooth - although maybe a little lumpy (the linear interpolation is an approximation to the real shape).

In the back row are cylinders with 12 faces. The face version clearly shows the sharp edges between the faces. The smooth-shaded one looks like a cylinder. This is true unless you look at it from the top, where the shape is clear.

Creating uneven surfaces

Here’s an example where we will control normals to create a stronger effect. It requires some explanation…

texturedemos/normals/n2-wavy.html

Start with the “wavy” object on the right. It is actually a white material; however, the lighting is set such that a strong red light comes from the right, and a strong blue light comes from the left. Because each surface points towards one light or the other, we see each face as one color or the other.

With the object in the middle, we used the normals from the wavy object (so it reacts to the light as if it were wavy). But it is flat.

On the left is the same flat object with the wavy normals. However, here I’ve used the “VertexNormalHelper” object to show what is happening.

The lesson here: The way that an object reacts with light is based on its normal directions. If we change the normal direction, we can fake the shape. We’ll come back to this idea later.
