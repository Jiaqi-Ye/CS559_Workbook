Page 19: Advanced Texturing

So far, we’ve focused on the simple form of texturing:

  1. We use the texture for color.
  2. We assign texture coordinates as uv values.
  3. We have an image that we loaded.

More advanced forms of texturing will generalize from these assumptions. They will allow us to create a wide range of surface effects, and they will provide us with tools to “fake” aspects of lighting that we are missing in the local lighting model.

The advanced forms of texturing are tricky to implement. Fortunately, the ones we will look at are built into three.js. The goal in exploring the texturing methods is to help you appreciate what they might be used for, and when you might want to try them.

The kinds of texturing that are important:

  1. Property Maps - use texture maps to control things other than color.
  2. Bump, Normal and Displacement Maps - use texture maps to make small changes to the apparent shape of the surface. Bump and Normal maps only change the directions of the normal vectors, causing the surface to interact with light as if it had changed. Displacement Maps actually move the surface, but require a different implementation.
  3. Environment Maps - use a color map to encode the colors in the world, not the colors on the object. This allows us to use texturing to simulate mirror reflections. A Skybox is a technique that uses the environment map to show the world around the viewer as a picture.
  4. Dynamic Environment Maps - have the system render the texture maps each frame, based on the world. This allows us to make reflections of moving objects, and other effects.
  5. Light Maps - use texture maps to simulate complex lighting.
  6. Layered Maps - combine multiple texture maps to achieve different effects.
  7. Shadow Maps - use texturing to simulate shadows.

We won’t learn about all of these in this workbook, but we’ll get through several. We might save some for the next workbook.
