Page 2: Practice: Cars

To review what we’ve done, we’re going to start with an exercise: make a vehicle (or two).

The pedagogical goals of this exercise are to give you practice defining shapes with meshes and adding textures to them. and gain more practice with the class framework. A reminder: the framework’s documentation is in its comments, but those comments are compiled into a more readable format here.

In the future, you will make Graphics Town, and you will probably want vehicles inside of the class software framework.

So on this page, please make a vehicle. It has to have wheels on the ground. Other than that, there are few restrictions; you can use a primitive (like a box or something) for the body. The vehicle must either have a distinct shape or texture to make it look like a car/truck/train.

If you load a model, it must be one that you made. Do not simply find a car model on the web.

Here are some simple ones from the 2000 Graphics Town I mentioned. You should be able to make something nicer.

To get points for this page, you must make a vehicle in the class software framework. That is, it must be defined as a GrObject. You can make 1 simple car. Or you can make something fancier. Don’t worry about making it drive around (that’s a future workbook). You can use this as an opportunity to explore how to make shiny materials. Or not.
