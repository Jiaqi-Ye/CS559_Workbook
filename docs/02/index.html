If we think of curves as something you can draw with a pen, we can use this “process” (of the pen moving over the curve) to define the curve. The idea of a parametric representation is that we use this “free parameter” (time) to index the set of points. We define a curve by a function that maps from time to a position in space.

There is a second piece to the definition: we need to know the range of times that we can ask positions about. Without loss of generality, we can always describe the range as going from beginning (0) to end (1), which is quite convenient. Sometimes we might want to use a different range, but we can scale and shift the range accordingly.

It is often convenient to have the parameter range over the unit interval from 0 to 1. When the free parameter ranges over the unit interval, we often denote it as u (to remind us that this is a unit parameterization). This is a convention, but I will try to use the convention consistently. If you see a different parameter, it may mean something else.

Parametric Curves and Parameterizations

A parametric curve (in 2D) is a shape represented by a function f(u), which returns 2D values. We can think of it as either x=f(u) or as separate functions for each dimension, x=fx​(u),y=fy​(u).

The parameter u of this function is the “free parameter” and it spans some range. The curve maps the interval (the range of the parameter) to a set of points in 2D.

Here is an example: the horizontal line segment from (0,0) to (1,0):

f(u)=(u,0)
(1)

Hopefully, this makes sense: put in u values, and get out points on the line segment. If we put in “all” the times, we get “all” the points (the infinite set).

Imagine tracing the same line segment twice. The first time, you go slowly at an even speed; the second time, you start slowly and speed up. Or maybe you draw right to left rather than left to right.

Notice that all three dots trace over the same line segment over the unit interval. Since we define the curve as the set of points traced, all three ways to draw the line segment are equivalent. We call these different parameterizations of the curve.

Aside: Mathematical Definitions

When I went looking, I found that mathematicians are not consistent in how they define curves.

In some cases, they define a curve as “the continuous image of some interval” or the set of points that are drawn. In this case, we would say that different parameterizations are of the same curve.

In other cases, they define curves as “A continuous map from a one-dimensional space to an n-dimensional space,” or the actual process of doing the mapping via the function. In this case, they would consider different parameterizations to be different curves.

I prefer the nomenclature where the curve is the set of points (the shape), so we speak of different parameterizations of the same curve. But in some places, this might get us in trouble. In those cases where it matters, I’ll be specific.

If you’re curious, here are the functions for the different parameterizations of the line segment. (In terms of the unit line segment; in the code, I have transformed it to put it in the right place in the Canvas).

fred​(u)fgreen​(u)fblue​(u)​=(u,0)=(u∗u,0)=(1−u,0)​
(2)

Notice that all of them map the interval [0,1] to the line segment [(0,0), (1,0)].

This is a boring function, but hopefully it makes the point that with a parametric representation we define curves as a function of the parameter, and that this is not unique. We can have multiple parameterizations.

Drawing Some Curves

Part of the reason we like parametric representations in computer graphics is that they are easy to draw. All we need to do is try a few different values of u, and connect the dots. Sometimes, it’s a little trickier than (since we need to know how many values to try, and how to connect them). But the simple strategy often works well.

You can see how the curve is “drawn” as u proceeds from 0 to 1. The drawing motion isn’t always important, if all we care about is the resulting thing that is drawn.

For each curve, we simply need to define some function f(u) that provides a 2D point for each value.

This simple example is important: notice how the curve “switches” at a u value. For each piece, it is well-behaved. But we may need to consider the switching point as a special case. The next example is connecting pieces from two circles (two arcs). It is similar.

The last curve - the two line segments on the right - is a single curve; it is just not connected. The pen “jumps” at .5 (look at the way the function is defined). This is very similar to the V shape, except that rather than just changing direction at .5, the pen jumps to a different spot.

Now we can look at the plotter function. It draws a curve - the function for the curve is passed as an argument. The parametric function is passed into the plotter via its func argument. This allows the plotter to draw any parametric function whose input ranges from 0 to 1. Notice that given a u value, each parametric function returns a list of two values for the position (x,y).

There is a slight catch that when the curves have discontinuities, we are careful about how we draw them. The code draws by “connecting the dots”, so it should only connect the dots that should be connected. And, the idea of drawing by taking uniform steps in u is very simple: in practice, we might want to choose the u values strategically (we’ll get to that later).

If you look at the plotter function, you can see that these curves are actually being drawn by approximating them by a number of line segments. Canvas is actually capable of drawing all of these shapes directly, but our code emphasizes the idea that we can draw parametric curves by sampling a bunch of parameter values. We’ll return to this idea later in the workbook.

In this program, we are drawing the curves by choosing parameter values with even spacing, but this means that we need to know how many samples to use when drawing the curve. For line segments, we only need two samples: u=0 and u=1, representing the endpoints of the line segment. However, for the smoother shapes, we use many segments (the circle uses 50) so that the shape looks as if it were a continuous circle. If you look carefully, you’d see it’s really a 50-sided polygon.

For curves where there is an abrupt change (such as the abrupt change in direction in the V or the jump in position in the two lines on the right), we need to be careful to choose values around the jump. In fact, for the two lines, we are not using the plotter code (which assumes that things are connected). We will call curves that exhibit a sudden change discontinuous. We’ll discuss continuity and discontinuity on a future page.

Drawing Curves in Practice

Drawing a curve by sampling a bunch of u values will turn out to be an inefficient strategy: we need to figure out how many u values to choose to convey the curve. If we draw too few, it won’t look “curvy”; if we draw too many, it is wasteful. We want to base the number of curve points on the number of pixels: enough points so there are no gaps, but nothing wasted by drawing the same pixel many times.

For many kinds of curves, there are highly efficient algorithms for drawing them “pixel perfect.” Unfortunately, we won’t cover them in class. But, we know that the API (or even the graphics hardware) can do this. Therefore, the right strategy for drawing curves efficiently is (almost always) to try to convert them to the kinds of curves that the API knows how to draw efficiently. These include straight line segments, elliptical arcs (affine transformations of circles), and Bezier curve segments (we’ll explain those in a bit).
