Workbook 5: Meshes, Texture and Appearance

This week, we’ll move on to making the objects that we see in 3D worlds. We’ll learn about how we group triangles into meshes and use texturing to give them more interesting colors. Along the way, we’ll see how shading is computed to make lighting effects, and examine different kinds of effects we can use to make things look more interesting.

Module Goals

In this workbook, we dig deeper into 3D graphics by exploring the techniques that we use to make shapes and appearance.

We will continue to explore using these methods in the context of a high-level API (three.js). Three implements all of the methods that we will learn about, and makes them easy to use. So this workbook is a mix of learning about the concepts, trying them out in simple cases, and learning a little bit about how they work “under-the-hood.” Understanding the concepts and a little about how they work is critical because these intuitions will help you know when to apply the methods.

  1. Understand meshes and use mesh data structures for describing objects.
  2. Practice the basic concepts of textures, especially texture coordinates.
  3. Understand how lighting and texturing are used to create appearance in interactice graphics systems.
  4. Practice using shape, material, and texture to create appearance.
  5. Understand how local lighting is computed and used in a practical system.
  6. Understand and apply advanced texturing types.

Exercises

There are a few “bigger” programming assignments scattered throughout the workbook:

  • Graphics Park - explore animation in the class software framework by making amusement park rides.
  • Construction Equipment - explore articulated figures and automatic user interfaces in the class software framework by making construction equipment.
  • Real Object - create an object that you see in the world by photographing textures.
  • Buildings - practice using textures by making a building.
  • Vehicles - practice using shape and appearance models by making a vehicle. (moved to another workbook)

There are several smaller exercises to expose you to using the ideas in three.js. These are still important requirements. This list is not exhaustive. Look at the rubric on Page  28  (Last Page (rubric)) for the complete list.

Readings

The workbook and videos should be self-contained. But, they don’t get as deep into the concepts as the books do. The readings are optional.

Accessing the Textbook(s)

I refer you to two textbooks, for different purposes.

If you want to learn the concepts, approached differently: Historically we used: Fundamentals of Computer Graphics. The book has a long author list (I contributed a chapter).

You can access the book from the UW Library (so you don’t need to buy it): Library Link.

I have also provided the book in Canvas.

If you want to dig deeper into the concepts and see the range of methods that get used in practice: I’ve recommended Real-Time Rendering by Thomas Moller, Eric Haines, Naty Hoffman, and some others (each edition adds authors). This book is 1200 pages, and they left chapters out! (the webpage gives PDFs for these chapters).

The most recent 4th edition is already old (it’s from 2018). They do continue to update the website.

This book is an amazingly extensive reference book that surveys many approaches and details for a huge variety of topics. It’s an invaluable reference - if you’re going to be a real-time graphics programmer, you probably want to have this book. For a class, it’s overkill - it’s way too much depth. It’s also not the best way to learn the concepts: once you understand the concepts, you can learn about a lot of details and different implementation tricks.

The UW Library has an electronic copy of Real-Time Rendering, 4e (Real-Time Rendering 4e (library link). Because the services the library uses are unreliable (and inconvenient when they work), I have provided some chapters in Canvas. I don’t have all of the chapters.

These reading recommendations refer to Fundamentals of Computer Graphics (FCG) and Real-Time Rendering (RTR) - described in the aside above.

Shading: FCG Chapter 10 covers the basics. (10.3 covers stylized shading, which we will not cover in class). If you want to dig deeper into lighting, RTR Chapter 5 is “Shading Basics” (but goes way beyond the basics).

Texturing: FCG Chapter 11 is concise yet thorough. RTR Chapter 6 covers even more details and specifics.

Meshes: FCG Chapter 12 discusses many different aspects of data structures to represent graphics. Sections 12, 12.1 and 12.2 cover the material for class. Section 12.1.3 covers strips and fans (which we will touch on in class, but won’t use much). Section 12.1.4 discusses the fancy data structures used for representing meshes when they need to be manipulated - this is beyond what we will do in class - read this section to get a taste of a more advanced topic.

Three.js: You will undoubtedly need to look at the THREE documentation. You may also want to look at the framework code documentation. The documentation is generated from the comments in the code, so you can probably just get by with reading the code.

A note on the assignments

It is your responsibility to make sure that your scene shows off the texturing feature that it is supposed to. For example, if you are showing off the use of environment maps to simulate reflections, make sure that you have a shiny object that reflects things (and an environment map that gives something to be seen in the reflections). You may want to make your objects move in order to show off some of the lighting effects, or choose object views that make things clear (for example, in lecture I showed off when bump mapping becomes invisible). If you think that it’s not clear that you are using the required effect, you may want to describe (in the text box below the exercise) what we should be seeing and why it shows off the feature.

You will need to find/create your own texture maps. Over the course of the assignment, you will need a bunch of texture maps. Over the whole assignment you must use each of these creation methods at least once:

  1. A texture you made yourself by painting (digital painting, using a painting program/photo editor)
  2. A texture that you got from the web (document where you found it, and confirm that it is free to use)
  3. A texture that is a photograph (that you took) of a real object

You need to explain how you used each of these creation methods on the last page of the workbook to get credit.

Don’t forget to include the textures you use in your repo and push them to GitHub!

If you want to load in objects from model files, that is OK (but optional). Make sure to include the objects in your repo, and document where you got them from in the text areas on the workbook pages. You probably want to use the loader code built in the framework. Please choose objects that have appropriate licensing and be sure to give proper attribution.

A Note on Textures

Please be reasonable with your textures. Because many students are uploading files for grading, it uses up a lot of space very quickly. You don’t need massive texture files. When you find textures on the web (especially environment map textures) they tend to be huge. Please choose small-sized textures, or shrink them to reasonable size if need be. (It is very unlikely that you need a texture that is more than 1MB).

And please use compressed file formats (e.g., .png or .jpg). Use HDR formats (such as .exr) only when necessary.

You do need to add/commit textures to your repo, so they get pushed to GitHub.

If you get a texture from someplace, you need to give proper attribution. Use the type-in box on Page  28  (Last Page (rubric)).
